# Security Policy

## Supported scope

Hellox is currently developed as a local-first Rust workspace. Security reports are
most helpful when they include:

- affected crate or command surface
- version / commit
- reproduction steps
- expected impact
- logs, traces, or proof-of-concept when safe to share

## Reporting a vulnerability

Please do not open a public issue for sensitive security reports.

Instead:

1. Use GitHub private vulnerability reporting if it is available for this repository.
2. If private reporting is unavailable, contact the repository maintainer through GitHub.
3. Share only the minimum proof needed to reproduce and assess the issue.

## Preferred report format

- Summary
- Affected file, crate, or command
- Reproduction steps
- Impact and attack assumptions
- Suggested mitigation, if known

## Response expectations

The project will try to:

- acknowledge valid reports
- reproduce the issue
- prepare a fix or mitigation when feasible
- avoid publishing sensitive details before a fix is available
